## 功能說明

此系統分成兩部分：

1. **表格公式**
   - 將「總表」連結到「學員名單」

   - 查詢結果會顯示在後方欄位

2. **App Script 網頁應用程式**
   - 提供報到頁面
   - 紀錄報到時間
   - 回填「學員報到」工作表的 A、B 欄

  - 標題列名稱必須與「學員名單」完全一致
  - `C2` 的公式請勿刪除

# 學員報到系統（Google Sheets + Apps Script）

這份檔案分成兩部分：

1. **表格公式**：把「學員報到」自動連到「學員名單」，輸入學員代號或手機後，顯示對應資料。  
2. **Apps Script 網頁應用程式**：提供報到介面，寫入報到紀錄時間到「學員報到」A、B 欄。

---

## 工作表需求

### 1) 學員名單

- 標題列（第 1 列）**必須包含**：
  - `學員代號`
  - `手機`
- 其餘欄位可自由擴充（例如：姓名、班級、序號...）。
- 標題列名稱會被「學員報到」拿來對應資料，建議命名固定、不隨意改動。

### 2) 學員報到

- A、B 欄由 Apps Script 自動填入：
  - A 欄：報到輸入值（學員代號或手機）
  - B 欄：報到時間
- **只有 `C2` 保留公式**（不可刪除）。
- 除了 `C2` 外，其餘儲存格都可編輯。
- 第 1 列標題可改；系統會依標題到「學員名單」抓對應欄位。
- 注意事項：
  - 標題列不能重複
  - 標題必須與「學員名單」標題完全一致（含文字）

---

## 功能說明

### 表格公式層

- 「學員報到」透過 `C2` 公式，依 A 欄輸入值（學員代號/手機）查詢「學員名單」。
- 查到後，將資料顯示在 C 欄以後對應欄位。
- 若查無資料，該列顯示空白（或依你目前公式設定顯示錯誤訊息）。

> `C2` 公式是核心，請勿刪除。

### Apps Script 層

- 提供前端報到頁（Web App + 側邊欄）。
- 報到時會呼叫 `checkInStudent()`，寫入「學員報到」新資料列：
  - A 欄：輸入值
  - B 欄：`new Date()` 報到時間
- 查詢邏輯同時支援：
  - `學員代號` 精確匹配
  - `手機` 精確匹配

---

## 額外選單（試算表上方）

開啟試算表後，會出現「`額外選單`」：

- `顯示報到側邊欄`：開啟學員報到系統 UI（Sidebar）。
- `備份公式`：備份整份試算表所有公式到 Script Properties。
- `還原公式`：從 Script Properties 還原公式（需先做過備份）。

---

## 複製與部署指引

1. **檔案 -> 建立副本**  
   會連同 Apps Script 一起複製。

2. 在新副本設定「學員名單」資料來源  
   - 依你的設計，先允許存取（通常會在 `A1` 觸發授權）。

3. **擴充功能 -> App Script -> 部署 -> 新增部署作業 -> 網頁應用程式**  
   - 執行身份：`我`  
   - 誰可以存取：`所有人`  
   - 點 `部署` 並完成授權。

4. 開啟 Web App 網址並點擊 **儲存目前 ID**  
   系統會把部署 ID 寫入 Script Properties，供側邊欄顯示 QR Code 連結使用。

---

## 常見錯誤排查

- 找不到學員：
  - 先確認「學員名單」有 `學員代號`、`手機` 標題。
  - 確認輸入值與名單內容是「完全一致」。
- 報到有寫入但後方欄位沒顯示：
  - 確認 `C2` 公式仍存在。
  - 確認「學員報到」標題沒有重複、且能在「學員名單」找到同名標題。
- 側邊欄沒有 QR Code 或連結：
  - 重新部署 Web App。
  - 再次開啟網址並按一次「儲存目前 ID」。
- 公式誤刪：
  - 從「額外選單 -> 還原公式」復原（前提是先做過備份）。

## 備註

- 目前選單函式名稱為 `backupFomula` / `restoreFomula`（程式內拼字沿用既有命名）。
- 若要升級多人同時報到穩定性，系統已使用文件鎖（`LockService`）避免同時寫入衝突。
